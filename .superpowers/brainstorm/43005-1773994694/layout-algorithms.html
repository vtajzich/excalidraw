What layout style should the engine use?

This determines how elements are arranged when you call the layout tool

A

Hierarchical (top-down / left-right)

Parent boxes contain or sit above children. Great for org charts, system diagrams, service trees. You define parent→child relationships and the engine places everything in tidy rows/columns.

B

Flow / Directed graph

Elements are arranged by the direction arrows flow. Best for pipelines, data flows, process diagrams. The engine follows arrow direction to decide left→right or top→bottom ordering.

C

Both — metadata-driven

The layout style is passed as metadata with each call. You can run hierarchical on one group and flow on another. The engine picks the algorithm based on what you tell it.

apply_layout({
algorithm: "hierarchical" | "flow",
direction: "top-down" | "left-right",
nodes: [...],
edges: [...]
})
