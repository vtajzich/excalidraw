Layout Engine Internals

How apply_layout processes a call — 4 phases in sequence

Processing Pipeline — src/layout.ts
1
Fetch & validate elements
Fetch all canvas elements (or subset from elementIds). Cross-reference with nodes[] metadata — fill in missing widths/heights from actual element dimensions. Validate that all parentId references exist in the set.
↓
2
Dagre layout (per containment group)
Build a Dagre graph from nodes[] and edges[]. Nodes with a parentId are processed as sub-graphs — Dagre runs on each group independently, then groups are positioned relative to their parent.
Hierarchical mode
rankdir: TB or LR
Sugiyama layering
crossing minimisation
node ordering per rank
Flow mode
same algorithm
edges drive direction
sources placed first
sinks placed last
Output: { id, x, y } for every node (absolute coordinates)
↓
3
Containment resolver (bottom-up)
Walks the parent-child tree leaves-first. For each parent: computes bounding box of all positioned children, adds padding on all sides, sets parent width/height to fit. Children's absolute positions are then offset to sit inside the parent box.
parent.x = minChildX - padding
parent.y = minChildY - padding
parent.width = (maxChildX - minChildX) + 2*padding
parent.height = (maxChildY - minChildY) + 2*padding
↓
4
Arrow routing
For each edge in edges[]: compute the nearest edge midpoints on source and target elements. Cast a ray between them — if it intersects no other element bounding box, use a straight 2-point line. If blocked, route as an elbow: go horizontally to midpoint X, then vertically to target Y (or vice versa, picking whichever path has fewer intersections).
Straight path
points: [[0,0], [dx,dy]]
gap: 8px from edge
updates startBinding
+ endBinding
Elbow fallback
points: 3-4 waypoints
exits source from nearest clear side
enters target perpendicularly
elbowed: true
Finally: batch-update all changed elements via Express API in one call
