Tool API Design

Three new MCP tools — review each and flag anything that looks off

1. apply_layout

apply_layout({
algorithm: "hierarchical" | "flow", // required
direction: "top-down" | "left-right", // default: top-down
elementIds: string[], // optional — omit to layout all
nodes: [{ // metadata for each node
id: string, // element ID on canvas
parentId?: string, // for containment — parent expands to hold this
width?: number, // hint — falls back to actual element width
height?: number
}],
edges: [{ // arrows to route after layout
fromId: string,
toId: string,
arrowId?: string // existing arrow to update, or creates new one
}],
spacing?: { // all optional, have sensible defaults
nodeSep?: number, // px between sibling nodes (default 40)
rankSep?: number, // px between layers/ranks (default 60)
padding?: number // px inside parent boxes (default 20)
}
})
// returns: { updated: number, positions: {id, x, y, width, height}[] }

2. move_element

move_element({
id: string, // element to move
x: number,
y: number,
arrowIds?: string[] // override: only reroute these arrows
// omit = auto-detect all connected arrows
})
// returns: { element: updated element, arrows: updated arrow[] }

3. create_arrow

create_arrow({
fromId: string, // source element
toId: string, // target element
label?: string, // text on the arrow
style?: "solid" | "dashed" | "dotted", // default: solid
startArrowhead?: "arrow" | "dot" | "bar" | null,
endArrowhead?: "arrow" | "dot" | "bar" | null, // default: arrow
color?: string // stroke color
})
// returns: { id: string, element: created arrow element }
// routing: straight if path is clear, elbow if blocked

Do these APIs look right? Any parameters missing or wrong? Reply in the terminal.
